  <action>
Create `playwright.config.ts` at project root with this exact configuration:

```typescript
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: './tests',
  retries: 0,
  workers: 1,
  reporter: [['html', { open: 'never' }], ['list']],
  webServer: {
    command: 'python3 -m http.server 8080',
    url: 'http://localhost:8080',
    reuseExistingServer: !process.env.CI,
    stdout: 'ignore',
    stderr: 'pipe',
  },
  use: {
    baseURL: 'http://localhost:8080/wolfond-report-2024-2026.html',
    trace: 'on-first-retry',
  },
  projects: [
    {
      name: 'desktop-chromium',
      use: {
        ...devices['Desktop Chrome'],
        viewport: { width: 1280, height: 800 },
      },
    },
    {
      name: 'mobile-iphone14',
      use: { ...devices['iPhone 14'] },
    },
    {
      name: 'mobile-pixel5',
      use: { ...devices['Pixel 5'] },
    },
  ],
});
```

Key decisions already locked:
- `workers: 1` — single server, avoid port conflicts
- `retries: 0` — deterministic tests, no flaky allowance
- `reuseExistingServer: !process.env.CI` — reuses running server locally, strict in CI
- `reporter: html + list` — html for local debugging, list for CI
- `trace: 'on-first-retry'` — traces only on failure retry (but retries=0, so effectively off unless changed)
- Device projects use Playwright registry device objects (`...devices['iPhone 14']`, `...devices['Pixel 5']`) — do NOT manually override viewport dimensions. The registry objects include correct UA, isMobile, hasTouch, deviceScaleFactor.
- Desktop Chromium uses `...devices['Desktop Chrome']` with viewport override to 1280x800.

Update `.gitignore` — append these lines to the existing file (do NOT overwrite existing entries):

```
# Playwright / npm
node_modules/
test-results/
playwright-report/
blob-report/
```

Also create an empty `tests/` directory by placing a `.gitkeep` in it (or skip — it will be created by Plan 02/03 when spec files are written).
