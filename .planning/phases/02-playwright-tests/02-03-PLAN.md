  <action>
Create `tests/mobile-viewport.spec.ts` with a `test.describe('Mobile viewport')` block. This spec runs on the mobile projects (mobile-iphone14, mobile-pixel5) to verify Phase 1 CSS fixes are applied.

Use `test.beforeEach` to navigate and wait for `.slide.active`.

Tests to implement:

1. **Active slide has touch-action: pan-y** ŌĆö Assert `.slide.active` has CSS `touch-action` equal to `pan-y` using `toHaveCSS('touch-action', 'pan-y')`.

2. **Active slide has overscroll-behavior-x: contain** ŌĆö Assert `.slide.active` has CSS `overscroll-behavior-x` equal to `contain` using `toHaveCSS('overscroll-behavior-x', 'contain')`.

3. **Nav dot ::before pseudo-element touch target >= 44px** ŌĆö Use `page.evaluate()` to read the computed height of `.slide-dot::before`:
```typescript
const height = await page.evaluate(() => {
  const dot = document.querySelector('.slide-dot');
  return parseFloat(getComputedStyle(dot!, '::before').height);
});
expect(height).toBeGreaterThanOrEqual(44);
```
Also check width >= 44px using the same pattern.

4. **No horizontal overflow on active slide** ŌĆö Assert `document.documentElement.scrollWidth <= document.documentElement.clientWidth` via page.evaluate. This confirms the mobile viewport is not wider than the screen.

5. **Slide inner fits within viewport** ŌĆö Get the bounding box of `.slide.active .slide-inner` and assert its width does not exceed the viewport width. Use:
```typescript
const box = await page.locator('.slide.active .slide-inner').boundingBox();
const viewport = page.viewportSize()!;
expect(box!.width).toBeLessThanOrEqual(viewport.width + 1); // 1px tolerance
```

6. **Content-heavy slide scrolls vertically** ŌĆö Navigate to a slide with `.scroll-content` (e.g., the Team slide). Use `page.evaluate` to find a slide index that contains `.scroll-content`:
```typescript
const scrollSlideIndex = await page.evaluate(() => {
  const slides = document.querySelectorAll('.slide');
  for (let i = 0; i < slides.length; i++) {
    if (slides[i].querySelector('.scroll-content')) return i;
  }
  return -1;
});
```
Navigate to that slide, wait 850ms. Assert the `.scroll-content` element has `overflow-y` set to `auto` or `scroll` (check both values ŌĆö use `page.evaluate` to read the computed value and assert it is not `hidden` or `visible`).

Important notes:
- Do NOT use `test.skip` to skip this spec on desktop ŌĆö instead rely on the `testMatch` or `testIgnore` config, OR simply run it and let the CSS assertions pass/fail on their own viewport. The simplest approach: these CSS rules are applied via media queries on mobile viewports, so assertions will naturally validate the correct behavior. If the project config sends this spec to mobile projects only via `testMatch`, that is handled by the executor in Plan 01's config. If not, the tests should still pass on desktop because the CSS rules are applied unconditionally (confirmed in Phase 1 ŌĆö touch-action and overscroll-behavior-x are not behind media queries; they are on `.slide` globally).
- Use `expect(height).toBeGreaterThanOrEqual(44)` not `toBe(44)` ŌĆö the target could be slightly larger.
