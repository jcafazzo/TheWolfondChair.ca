Selectors:

- OR use counter text assertion which auto-retries until transition completes

CRITICAL — Slide Count:

- Tests MUST read `slides.length` at runtime, not hardcode 17
- Counter format will be `NN / 18` (or whatever actual count is)

  <name>Task 1: Create navigation.spec.ts</name>
  <files>tests/navigation.spec.ts</files>
  <action>
Create `tests/navigation.spec.ts` with a `test.describe('Navigation')` block. Use `test.beforeEach` to navigate to the page and wait for `.slide.active` to appear (the app builds slides dynamically on DOMContentLoaded — without this wait, locators will time out).

Tests to implement:

1. **ArrowRight advances slide** — Press ArrowRight, assert `#slideCounter` text becomes `02 / {total}` (where total is read at runtime via `page.evaluate(() => slides.length)`). Also verify `window.current` is 1 via page.evaluate.

2. **ArrowLeft retreats slide** — Navigate to slide 2 first (press ArrowRight, wait for counter to show `02 / {total}`), then press ArrowLeft, assert counter shows `01 / {total}` and current is 0.

3. **Dot click navigates to target slide** — Click `.slide-dot` at index 4 (nth(4)), assert counter shows `05 / {total}`. Verify the clicked dot has class `.active` using `toHaveClass(/active/)`.

4. **Arrow button click (next) advances slide** — Click the second `.slide-arrow` (nth(1) — next button), assert counter updates. Click the first `.slide-arrow` (nth(0) — prev button), assert counter returns.

5. **All slides reachable — forward traverse** — Read total from `slides.length`. Loop: press ArrowRight, wait 850ms for transition. After loop, assert `window.current` equals `total - 1`. This is the slow test — set a generous timeout with `test.slow()` to allow 3x the default timeout.

6. **Boundary: cannot go before first slide** — On slide 0, press ArrowLeft. Assert `window.current` is still 0 and counter still shows `01 / {total}`.

7. **Boundary: cannot go past last slide** — Navigate to last slide via `page.evaluate(() => goToSlide(slides.length - 1))`, wait 850ms. Press ArrowRight. Assert current is still `total - 1`.

Important implementation notes:
- Use `page.keyboard.press('ArrowRight')` (not `page.press('body', ...)`).
- For counter text assertions, use `await expect(page.locator('#slideCounter')).toHaveText(...)` — this auto-retries and handles transition timing.
- For dot active class, use `await expect(page.locator('.slide-dot').nth(N)).toHaveClass(/active/)`.
- Read total slide count ONCE in beforeEach or at test start via `const total = await page.evaluate(() => slides.length)` and build expected counter strings from it. Do NOT hardcode "17" or "18".
- For the traverse-all test, use `test.slow()` as it will take ~15 seconds (18 slides * 850ms).
