  <action>
Present the local server URL to the user and ask them to verify all Phase 1 fixes on their mobile device.

All Phase 1 mobile fixes have been applied:
- **CSS:** Viewport height uses 100svh with 100vh fallback (MCSS-01), touch-action: pan-y on slides (MCSS-02), 44px invisible dot touch targets (MCSS-03), overscroll-behavior-x: contain (MCSS-04)
- **JS:** Touch handler captures scroll element at touchstart (MJS-02, enabling MJS-01), Page Visibility API pauses/resumes auto-advance timer (MJS-03)

Ask the user to open `http://{local-ip}:8080/wolfond-report-2024-2026.html` on their phone and test these 5 checks:

**1. Viewport Height (MCSS-01):**
- Scroll up/down to trigger the address bar showing and hiding
- Verify: slide content does NOT shift/resize when the address bar appears or disappears
- Test on at least one iPhone if available

**2. Scroll vs Swipe (MJS-01 + MJS-02):**
- Navigate to a slide with scrollable content (publications list or team members)
- Swipe vertically inside the scrollable area
- Verify: content scrolls up/down WITHOUT advancing to the next slide
- Then swipe horizontally: verify the slide DOES advance

**3. Nav Dot Tapping (MCSS-03):**
- Tap on different nav dots along the right side
- Verify: dots respond to taps even if you tap slightly outside the visible 8px circle
- Verify: no visible ghost/highlight artifact around the dots

**4. Horizontal Overscroll (MCSS-04):**
- On any slide, swipe horizontally past the edge
- Verify: the browser does NOT navigate back/forward (on iOS 16+)
- Note: If testing on iOS 15 or below, browser-back may still fire — this is a known accepted limitation

**5. Timer Pause on Tab Hide (MJS-03):**
- Watch the auto-advance countdown timer at the bottom
- Switch to another browser tab or app for ~10 seconds
- Switch back
- Verify: the timer resumes from approximately where it was, not from 120s
- Verify: no slide jump happened while the tab was hidden

**Bonus — Desktop regression:**
- Check on desktop that keyboard arrows, mouse click on dots/arrows, and mousewheel still work as before

Wait for user response: "approved" or description of failures.
